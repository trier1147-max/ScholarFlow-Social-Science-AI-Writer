# ScholarFlow 产品需求文档 (PRD)

> 版本: 1.0 | 更新日期: 2026-01-20

## 1. 产品概述

### 1.1 产品定位
ScholarFlow 是一款专为人文社科领域研究生及科研人员打造的 AI 学术写作工作台。通过构建私有文献数据库，利用 RAG（检索增强生成）技术，帮助用户基于真实文献进行学术写作，实现"无据不言"的高质量原创输出。

### 1.2 核心价值主张
> "不是 AI 帮你写论文，是 AI 帮你**用好你自己的文献**来写论文"

### 1.3 目标用户
| 用户群体 | 特征 | 核心需求 |
|---------|------|---------|
| 硕士研究生 | 有一定文献积累，写作经验不足 | 快速产出课程论文、文献综述 |
| 博士研究生 | 文献量大，需要系统整理 | 高效利用文献，产出高质量论文 |
| 高校教师/科研人员 | 时间紧张，写作任务重 | 提升写作效率，润色打磨 |

### 1.4 使用场景
1. **课程论文**: 基于指定文献完成课程作业
2. **文献综述**: 梳理研究领域现状，发现研究空白
3. **期刊论文**: 撰写学术论文初稿，进行润色修改
4. **学位论文**: 辅助完成硕博学位论文各章节

## 2. 竞品分析

### 2.1 竞品对比
| 维度 | NotebookLM | Consensus | Jenni AI | **ScholarFlow** |
|------|------------|-----------|----------|-----------------|
| 核心功能 | 文献问答 | 学术检索 | 通用写作 | **私有库写作** |
| 输出形式 | 笔记/摘要 | 检索结果 | 文章段落 | **完整论文** |
| 引用来源 | 上传文献 | 公开论文 | 无明确来源 | **私有文献库** |
| 学科适配 | 通用 | 通用 | 通用 | **人文社科** |
| 引用回溯 | 有 | 有 | 无 | **有(精确定位)** |

### 2.2 差异化优势
1. **私有文献驱动**: 只基于用户自己的文献生成，确保原创性
2. **人文社科专精**: 针对批判性写作逻辑进行优化
3. **完整写作流程**: 从文献管理到成稿的一站式体验
4. **精确引用回溯**: 每个引用都可追溯到 PDF 原文位置

│                         ScholarFlow                              │

│   文献管理      │    AI 对话        │       写作编辑             │
│   Library      │    Chat           │       Editor               │
├────────────────┼───────────────────┼────────────────────────────┤

│ · 向量化存储    │ · 引用插入        │ · 版本管理                 │

└────────────────┴───────────────────┴────────────────────────────┘

### 3.2 功能模块详述

#### 模块 A: 文献管理 (Library)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| PDF 上传 | 支持单个/批量上传 PDF 文件 | P0 |
| 自动解析 | 提取正文、元数据（标题、作者、年份） | P0 |
| 向量化 | 将文献切片并向量化存储 | P0 |
| 文献列表 | 展示、搜索、筛选已上传文献 | P0 |
| 文献详情 | 查看文献元数据、摘要、章节结构 | P1 |
| 文献选择 | 勾选文献作为当前对话上下文 | P0 |
| 文献删除 | 删除文献及其向量数据 | P0 |

#### 模块 B: AI 对话 (Chat)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| RAG 问答 | 基于选中文献回答问题 | P0 |
| 写作生成 | 生成文献综述、论证段落等 | P0 |
| 流式输出 | SSE 流式返回生成内容 | P0 |
| 引用标注 | 生成内容自带脚注引用 | P0 |
| 引用回溯 | 点击引用跳转 PDF 原文 | P1 |
| 历史记录 | 保存对话历史 | P1 |
| 模式切换 | 严格模式/探索模式 | P2 |

#### 模块 C: 写作编辑 (Editor)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 富文本编辑 | 基于 Tiptap 的编辑器 | P1 |
| Markdown 支持 | 实时预览 Markdown | P1 |
| 内容发送 | 将 AI 生成内容发送到编辑器 | P1 |
| 选中润色 | 选中文字发送给 AI 润色 | P1 |
| 草稿保存 | 自动保存草稿 | P1 |
| 版本历史 | 查看历史版本 | P2 |
| 导出功能 | 导出 Word/PDF/Markdown | P2 |

### 3.3 多智能体架构

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    Agent Orchestrator                       │
└─────────────────────────────────────────────────────────────┘
    │
    ├──────────────────┬──────────────────┐
    ▼                  ▼                  ▼
┌──────────┐    ┌──────────┐      ┌──────────┐
│Researcher│    │  Writer  │      │  Editor  │
│ 检索证据  │───→│ 撰写初稿  │───→  │ 润色优化  │
└──────────┘    └──────────┘      └──────────┘
```

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **Researcher** | 从向量库检索相关文献片段 | 用户问题 + 文献范围 | 证据卡片列表 |
| **Writer** | 基于证据组织论证并撰写 | 证据卡片 + 写作指令 | 带引用的初稿 |
| **Editor** | 消除 AI 痕迹，优化学术语气 | 初稿文本 | 润色后的终稿 |

## 5. 技术方案

### 5.1 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Frontend (Next.js)                       │
└─────────────────────────────────────────────────────────────────┘
                              │ HTTP/SSE
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Backend (FastAPI)                          │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐                │
│  │ 文献服务    │  │ RAG 服务   │  │ Agent 服务  │                │
│  └────────────┘  └────────────┘  └────────────┘                │
└─────────────────────────────────────────────────────────────────┘
          │                │                │
          ▼                ▼                ▼
   ┌──────────┐     ┌──────────┐     ┌──────────┐
   │ SQLite   │     │ Chroma   │     │ DeepSeek │
   │ (元数据)  │     │ (向量)   │     │ (LLM)    │
   └──────────┘     └──────────┘     └──────────┘
```

### 5.2 数据模型

```
Document (文献)
├── id: UUID
├── title: string
├── authors: string[]
├── year: int
├── source: string (期刊名)
├── abstract: string
├── file_path: string
└── created_at: datetime

Chunk (文献切片)
├── id: UUID
├── document_id: UUID (FK)
├── content: string
├── embedding: vector
├── page_number: int
├── section_title: string
└── chunk_index: int

Conversation (对话)
├── id: UUID
├── project_id: UUID (FK)
├── role: 'user' | 'assistant'
├── content: string
├── citations: JSON
└── created_at: datetime
```

### 5.3 核心 API

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/documents/upload | 上传 PDF 文献 |
| GET | /api/documents | 获取文献列表 |
| DELETE | /api/documents/{id} | 删除文献 |
| POST | /api/chat | 发送对话消息 (SSE) |
| POST | /api/write/generate | 生成写作内容 |
| POST | /api/write/polish | 润色文本 |

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| PDF 解析质量差 | 引用不准确 | 提供手动编辑入口 |
| LLM 成本过高 | 无法持续运营 | 限制免费额度，优化 prompt |
| AI 痕迹明显 | 用户不满意 | 持续优化 Editor Agent |
| 版权问题 | 法律风险 | 仅存储用户自有文献 |
